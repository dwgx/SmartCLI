어떤 터미널 프로그램이든
사람처럼 조작.

사람이 AI에게 할 일을 지시합니다. 하지만 메뉴, 스피너, 비밀번호 입력란, 에이전트 CLI는 단순한 파이프에는 보이지 않습니다. SmartCLI는 모델이 화면을 인식하고, 키를 누르고, 결과를 확인하게 해줍니다 — 사람이 하는 방식 그대로.

증명 — 실제 TUI 조작

진짜로 lazygit을 조작합니다.

SmartCLI가 실제 lazygit TUI를 인식 → 조작 → 확인 루프로 직접 조작합니다 — 방향키로 패널을 이동하고, 커밋의 diff를 열고, 브랜치를 강조하는 것까지 모두 pyte 셀 그리드(어느 행이 선택됐는지, alt-screen 여부)를 읽어서 수행합니다. 실제 프로그램을 Linux 컨테이너에서 직접 조작해 캡처했고, 스크립트가 아닙니다. pexpect 같은 바이트 스트림 매처는 "어느 행이 강조됐는지"를 인식할 수 없지만, 화면 모델은 할 수 있습니다.

모든 클립은 SmartCLI가 실제 프로그램을 조작한 것입니다 — pyte 화면 그리드를 읽고 키를 누른 것이지, 스크립트로 만든 출력이 아닙니다. pexpect 같은 바이트 스트림 매처는 "어느 행이 강조됐는지"를 인식할 수 없지만, 화면 모델은 할 수 있습니다.

하나의 API · 세 가지 OS

같은 드라이버. Windows, macOS, Linux.

진짜로 어렵고, 방어 가능한 지점: 하나의 라이브러리가 세 플랫폼 모두에서 동일한 실제 lazygit을 동일한 인식 → 조작 → 확인 루프로 조작합니다 — Windows의 네이티브 ConPTY, macOS의 BSD pty, 그리고 Linux pty에서, 각각 적응형 DECCKM/SS3 방향키로. WSL도, Cygwin도 없습니다. 아래 각 클립은 해당 OS에서 실제로 조작한 것입니다.

Windows는 ConPTY에서 lazygit 0.63을 조작했고, macOS는 Apple Silicon M 시리즈에서 BSD pty로 조작했으며(그곳의 DECCKM/SS3 방향키 경로는 실제 ncurses에 대해 검증했습니다), Linux는 표준 라이브러리 pty.fork()로 조작했습니다. 같은 scripts/tui.py, 같은 recipes, 세 가지 OS.

시작하기

압축을 풀고, 넣기만 하면, 동작합니다.

코어는 PyPI에서 설치하고, 스킬은 그 자리에서 실행됩니다. 넣어둔 drive-tui 폴더는 자체 코어까지 함께 담고 있습니다.

# 공유 코어, PyPI에서 pip install smartcli-toolkit # 그다음, Python에서 from smartcli_core import PtySession # 또는 넣어둔 스킬 폴더가 제대로 연결됐는지 확인 python skills/drive-tui/scripts/tui.py doctor
